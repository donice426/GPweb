1. Introducción al software general

El software del sistema gpVision™ constituye el núcleo inteligente de las líneas de clasificación de GP Graders, diseñada para transformar las operaciones de empaque de productos frescos como cerezas, arándanos y frutas de hueso. Esta plataforma opera bajo una arquitectura cliente-servidor, donde la estación del operador (GUI) actúa como el cliente que controla los dispositivos y las Vision Units funcionan como nodos de procesamiento dedicados por carril.

Los pilares fundamentales de este ecosistema de software son los siguientes:

  • Interfaces de Usuario Especializadas: El sistema ofrece la Simple GUI para una operación diaria intuitiva basada en indicadores en tiempo real (producción, velocidad y carga), y la Operator Station para configuraciones técnicas profundas y ajustes de programas de calificación.
  • Análisis Óptico MultiView™: El software gestiona el escaneo del 100% de la superficie del producto, procesando imágenes en color y NIR (infrarrojo cercano) para eliminar la necesidad de clasificación manual.
  • Gestión Lógica y de Datos: A través de un motor basado en Microsoft SQL Server, el sistema registra estadísticas de producción cada minuto y permite la configuración de Defectos Modificables y mapeo de características especiales.
  • Control de Salidas (Exit Manager): El módulo GP Exit Manager se encarga de la logística física, supervisando los formatos de empaque, el peso acumulado y la distribución precisa del fruto hacia las salidas asignadas.
  • Innovación en Inteligencia Artificial: La plataforma está evolucionando hacia la integración de radiai™, una nueva generación de clasificación basada en algoritmos de aprendizaje profundo e IA.

Analogía para su comprensión: Si el hardware de la máquina es el cuerpo, el software gpVision™ es el sistema nervioso central. Mientras que las cámaras envían señales visuales (los ojos), el software procesa esa información en milisegundos para decidir el destino de cada fruto, actuando como el cerebro que coordina tanto el juicio de calidad como las órdenes motoras enviadas a los actuadores físicos.

En conjunto, Simple GUI funciona como el “tablero de control” operativo para el seguimiento continuo de la línea.

3. Acceso remoto a Vision Unit (Consola SSH / PuTTY)

Herramientas de acceso remoto utilizadas para la comunicación directa con las Vision Units del sistema gpVision™, orientadas a diagnóstico, mantenimiento y verificación del estado operativo.

Ventana de configuración de conexión remota (PuTTY)

Acceso seguro vía SSH a las Vision Units

Ventana de configuración de conexión remota (PuTTY) utilizada para el acceso directo a las Vision Units del sistema gpVision™. Esta herramienta permite la comunicación segura vía SSH para tareas de diagnóstico, mantenimiento y verificación del estado operativo de las unidades de procesamiento por carril.

Cada Vision Unit funciona como un nodo de procesamiento independiente encargado del análisis de imágenes MultiView™ en su respectivo carril. A través de una conexión SSH, los técnicos pueden acceder al sistema operativo interno de la unidad para revisar servicios activos, estados de red, consumo de recursos y registros de funcionamiento (logs).

El acceso remoto es una herramienta clave durante procesos de puesta en marcha, soporte técnico y resolución de fallas, ya que permite intervenir las Vision Units sin necesidad de acceso físico directo al hardware. Esto reduce tiempos de inactividad y facilita el soporte remoto por parte de GP Graders o personal técnico especializado.

Normalmente, este tipo de acceso está restringido a usuarios autorizados y se utiliza exclusivamente para tareas técnicas avanzadas, garantizando la seguridad y estabilidad del sistema de clasificación durante la operación productiva.

5. Configuración de clases y defectos gpVision™

Configuración del núcleo lógico del programa de calificación: definición de clases, umbrales y parámetros clave de clasificación.

Pantalla de configuración en Simple GUI

Ajuste de clases de calidad, defectos detectables y parámetros operativos

Pantalla de configuración de clases y defectos en la Simple GUI de gpVision™, donde se definen las categorías de calidad (Clase Superior, Comercial y Desechos) y los umbrales de defectos detectables. La interfaz permite ajustar parámetros como puntos blandos, tamaño, color y defectos combinados, constituyendo el núcleo lógico del programa de calificación y asegurando una clasificación consistente según los estándares definidos.

Esta pantalla cumple un rol central en la estandarización de la calidad, ya que traduce los criterios del proceso (por ejemplo, tolerancia a defectos o límites de tamaño) en reglas configurables para el sistema. Al estructurar la fruta en clases bien definidas, se logra una separación coherente que puede mantenerse estable durante toda la temporada o ajustarse por lote, variedad o condición de producto.

A nivel operativo, el ajuste fino de umbrales permite mejorar la consistencia del resultado: un valor demasiado estricto puede aumentar el rechazo (desechos), mientras que uno demasiado flexible puede degradar la calidad de la clase superior. Por eso, este módulo se usa frecuentemente junto con resultados recientes y validación visual para confirmar que el comportamiento del sistema coincide con los estándares del cliente.

La configuración también facilita la implementación de defectos combinados, lo cual permite agrupar condiciones que en la práctica se tratan como una sola regla de decisión. Esto ayuda a simplificar el control del programa, reducir ambigüedades y mantener una lógica de clasificación clara, especialmente cuando se requiere distinguir entre fruta premium, comercial y descarte.

10. Error de segmentación de fruta (Contorno azul incorrecto)

Ejemplo de detección incorrecta del contorno del fruto durante el proceso de segmentación en gpVision™.

Error de segmentación en imágenes MultiView™

Contorno detectado no coincide con la geometría real del producto

Ejemplo de error de segmentación en gpVision™, donde el contorno azul no coincide correctamente con la fruta real. Este desajuste se produce por una detección incorrecta del borde, generalmente asociada a descalibración del cutter, desfases de sincronización o condiciones de iluminación que afectan la separación entre fruta y fondo, impactando la precisión de las mediciones y la clasificación.

La segmentación correcta es un paso crítico del proceso de análisis, ya que define el área exacta del fruto sobre la cual se calculan métricas como tamaño, forma, volumen estimado y localización de defectos. Cuando el contorno detectado es incorrecto, estas mediciones pueden verse alteradas, provocando errores en la asignación de clases o en la evaluación de defectos.

Este tipo de error puede originarse por variaciones en la iluminación, acumulación de suciedad en componentes ópticos, cambios mecánicos en el sistema de corte (cutter), o desajustes temporales entre la captura de imágenes y el posicionamiento del fruto. Por ello, la revisión de estos resultados es fundamental durante tareas de diagnóstico y mantenimiento.

El módulo de resultados detallados permite identificar visualmente estos problemas, facilitando la toma de acciones correctivas como recalibración, ajuste de parámetros de segmentación o intervención técnica, con el fin de restablecer la precisión del sistema gpVision™ y asegurar una clasificación confiable y consistente.

12. Consola de red de Vision Unit

Herramienta de diagnóstico para verificar conectividad, estado de interfaces y tráfico de comunicación en las Vision Units del sistema gpVision™.

Conectividad Ethernet y tráfico de datos

Validación de subredes, enlaces activos y estabilidad de comunicación

Consola de red de una Vision Unit del sistema gpVision™, mostrando el estado de las interfaces Ethernet y el tráfico de datos en múltiples subredes. Esta vista se utiliza para verificar conectividad, carga de comunicación y correcto funcionamiento de la infraestructura de red que enlaza cámaras, Vision Units y el servidor central.

En esta consola se observan parámetros críticos como direcciones IP, estado del enlace (UP/DOWN), configuración de broadcast/multicast, conteo de paquetes transmitidos/recibidos y métricas de errores. Estos indicadores permiten detectar rápidamente pérdidas de paquetes, colisiones, caídas de enlace o saturación de tráfico.

Esta información es especialmente importante en arquitecturas donde una Vision Unit procesa imágenes por carril y debe enviar resultados en tiempo real hacia el servidor/estación del operador. Una degradación en la comunicación puede provocar retrasos, desconexiones, fallas de sincronización o carriles detenidos por seguridad operativa.

En procedimientos de soporte, esta vista se utiliza para confirmar que cada interfaz esté en la subred correcta, que exista ruta válida hacia el servidor y que el volumen de datos sea consistente con la carga de producción. También ayuda a diferenciar fallas de hardware (NIC, cable, switch) de problemas de configuración (IP, máscara, gateway, VLAN).

Este módulo actúa como una “bitácora técnica” del sistema, registrando de forma cronológica eventos críticos, advertencias repetitivas y estados anómalos asociados a componentes clave. Al concentrar el detalle por carril, timestamp y categoría (por ejemplo: cámaras), permite identificar rápidamente si el problema se limita a una Vision Unit, a un subconjunto de cámaras o a una condición general de la línea.

En escenarios de producción, estas advertencias pueden correlacionarse con síntomas como disminución de velocidad, pérdida de imágenes, retrasos en procesamiento o desconexiones intermitentes. Por eso, la revisión de Logs es un paso esencial antes de realizar ajustes de calibración o cambios en programas de clasificación, ya que ayuda a confirmar si la causa es lógica (configuración/software) o física (cableado, iluminación, sincronía, red).

Como práctica recomendada, si se observan advertencias repetidas en el mismo carril o cámara, se debe verificar primero la conectividad de red, la estabilidad del enlace hacia la Vision Unit, y luego revisar sincronización de disparo, condiciones de iluminación y estado del hardware asociado (cámara, trigger, encoder o switch).

20. Error de Software – “Unhandled error occured”

Ejemplo de error no controlado en gpVision™ durante la ejecución de un programa de clasificación, con impacto directo en la continuidad operativa del proceso.

Error no controlado en la interfaz

Este mensaje indica que el sistema encontró una condición no prevista y detuvo el proceso actual.

La interfaz muestra un “Unhandled error occured”, indicando que el software encontró una condición no prevista durante la ejecución del programa de clasificación. Este tipo de error suele producirse por una desincronización interna (por ejemplo, pérdida de comunicación con un módulo, datos incompletos del programa activo o un fallo temporal del servicio), lo que impide que el sistema gestione correctamente la excepción y detiene el proceso actual.

En términos prácticos, este mensaje suele aparecer cuando un componente crítico del flujo (adquisición de imágenes, servicios de análisis, comunicación con Vision Units, base de datos o módulos de salida) deja de responder o entrega información inconsistente respecto al estado del programa en ejecución. Al no existir una rutina de recuperación para esa condición específica, gpVision™ corta la operación para evitar resultados erróneos o decisiones de clasificación inválidas.

Para el diagnóstico, normalmente se recomienda revisar: (1) el panel de estado del sistema y los carriles/cámaras conectadas, (2) el registro de eventos (Logs) para ubicar el origen del fallo, (3) la conectividad de red entre estación de operador, servidor y Vision Units, y (4) la integridad del programa activo (parámetros de clases, defectos, calibres y salidas).

Si el error ocurre de forma recurrente en un mismo punto del proceso, suele indicar una causa persistente (desfase de sincronización, reinicios de servicio, datos corruptos o una configuración incompleta). En ese caso, la revisión técnica debe enfocarse en estabilizar el servicio afectado y validar que todos los módulos necesarios estén operativos antes de reiniciar la calificación.

22. Defecto en cavidad del pedicelo (NIR)

Detección de alteraciones internas en la zona del pedicelo mediante análisis en infrarrojo cercano (NIR).

Detección de defecto interno en cavidad del pedicelo

Análisis NIR aplicado a tejido interno no visible en color

La cereza presenta daño en la cavidad del pedicelo, donde el sistema detecta una alteración interna del tejido asociada a una herida o a un proceso de pudrición inicial. Este tipo de defecto no siempre es visible externamente en imágenes de color, pero se manifiesta claramente mediante el análisis en infrarrojo cercano (NIR).

En la imagen NIR, esta zona aparece con una respuesta anómala en comparación con el tejido sano circundante, indicando una pérdida de firmeza o una degradación interna del tejido. La detección de defectos en la cavidad del pedicelo es especialmente relevante, ya que esta zona suele ser un punto de ingreso para patógenos y puede evolucionar rápidamente hacia pudriciones más extensas durante el almacenamiento o transporte.

Gracias al uso combinado de imágenes MultiView™ y análisis NIR, gpVision™ puede clasificar estos frutos como defectuosos de forma temprana, evitando que productos con daño interno ingresen a categorías de mayor valor comercial y asegurando una clasificación más precisa y consistente.
